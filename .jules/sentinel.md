## 2024-05-18 - Missing Authentication Middleware on Sensitive Endpoints
**Vulnerability:** The `auth_middleware` function was defined in `crates/hl7v2-server/src/middleware.rs` but never applied to the protected `/hl7/parse` and `/hl7/validate` endpoints in `routes.rs`. Additionally, it relied on a non-constant-time string comparison (`key == expected_key`) and re-read the environment variable on every request, which is inefficient and insecure.
**Learning:** Defining middleware functions is insufficient; they must be explicitly bound to the router (e.g., using `.layer()`). Relying on environment variables directly in request hot paths can lead to state inconsistency and performance issues. Custom string comparison for secrets must use a constant-time algorithm to prevent timing attacks.
**Prevention:** Ensure that stateful authentication configuration is loaded once during server initialization into a centralized state (e.g., `AppState`) and passed to middleware via `axum::extract::State`. Apply authentication middleware explicitly to routers containing sensitive endpoints, and always use constant-time equality checks for secrets.
