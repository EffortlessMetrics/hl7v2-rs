# Security Policy

## Supported Versions

Only the latest stable release is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.2.x   | ✅ Yes              |
| < 1.2.0 | ❌ No               |

## Reporting a Vulnerability

We take the security of this project seriously. If you believe you have found a security vulnerability, please do **not** open a public issue.

Instead, please report it via one of the following methods:

1.  **Email**: Send a detailed report to `security@effortlessmetrics.com`.
2.  **GitHub Private Reporting**: Use the "Report a vulnerability" button on the GitHub Security tab if available.

### What to Include

- A detailed description of the vulnerability.
- Steps to reproduce (including a minimal HL7 message if relevant).
- Potential impact.
- Any suggested mitigations.

### Our Process

- We will acknowledge receipt of your report within 48 hours.
- We will provide an estimated timeline for a fix.
- Once a fix is verified, we will release a new version and credit you for the discovery (unless you prefer to remain anonymous).

### PHI / Sensitive Data

**Important**: When reporting vulnerabilities, please do **not** include any real Protected Health Information (PHI) or personally identifiable information. Use synthetic data generated by `just scaffold` or the `hl7v2-faker` crate.

Thank you for helping keep our healthcare infrastructure secure! 🛡️
